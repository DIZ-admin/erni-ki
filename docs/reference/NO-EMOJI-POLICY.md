# Политика запрета эмодзи

Применяется ко всем файлам репозитория (код, документация, конфигурации).

## 1. Цель

Гарантировать единообразный деловой стиль и корректное отображение во всех
редакторах без зависимости от шрифтов и платформ.

## 2. Правила

1. Эмодзи, пиктограммы, Unicode-символы из категорий «Misc Symbols and
   Pictographs» и «Emoticons» запрещены.
2. Допустимые символы — ASCII (0x20-0x7E) и стандартные знаки пунктуации.
3. Для маркировки статуса используем текст (`Status: PASS`) и markdown чекбоксы.
4. В таблицах статусов вместо символов используем текст `YES/NO/IMPORTANT`.

## 3. Исключения

- UI-мастер-классы и пользовательские инструкции, где нужно показать эмодзи из
  интерфейса внешнего продукта (только в кодовых блоках и с пометкой).
- JSON-ответы внешних API, где эмодзи — часть данных. Такие примеры помещаются в
  fenced code block и снабжаются примечанием.

## 4. Контроль

- pre-commit hook `no-emoji-in-files` блокирует коммиты.
- Для временного пропуска (например, текст тестовых данных) добавляется
  pragma-комментарий `<!-- pragma: allowlist emoji -->` и issue на удаление.
- За нарушение политики PR отклоняется до устранения.

## 5. Связанные документы

- `docs/reference/style-guide.md`
- `docs/reference/documentation-maintenance-strategy.md`
- `DOCUMENTATION_AUDIT_REPORT.md`

Поддерживайте текст в ASCII, чтобы материалы ERNI-KI корректно отображались во
всех CLI/IDE и соответствовали корпоративному стилю.
