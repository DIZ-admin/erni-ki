# Политика языков

1. **Код, конфигурации и скрипты всегда на английском**: комментарии, сообщения
   логов, ошибки и вывод CLI не могут содержать кириллицу или немецкие символы.
2. **Документация допускает только ru/de/en**: основное дерево `docs/` считается
   русским, `docs/de/` — немецким, `docs/en/` — английским. Новые файлы обязаны
   содержать `language: <locale>` во фронт-материи.
3. **Примеры локализованного контента** (например, status snippet) должны
   хранить текст в `docs/*` или отдельных JSON/YAML файлах, а не в исходниках.

# Автоматическая проверка

- Локально: `npm run lint:language` для проверок staged-файлов.
- Полная проверка: `node scripts/language-check.cjs --all`.
- CI (`.github/workflows/ci.yml`) запускает `npm run lint:language -- --all` и
  блокирует PR при нарушении правила.

# Baseline

Файл `language-policy.config.json` содержит список исторических файлов, в
которых ещё осталось русскоязычное содержимое. Скрипт выводит их в блоке
`Language policy baseline`, чтобы Contributors могли постепенно переводить их на
английский. Добавляйте файл в baseline только если:

- По задаче согласовано постепенное исправление.
- В PR приложен план перевода.

После перевода файла базовый список необходимо обновить (удалить путь) и
запустить `node scripts/language-check.cjs --all`.

# Чек-лист перед PR

1. Переведены все комментарии/сообщения в изменённых `.ts/.js/.sh/.py/.yml`.
2. Тексты, которые должны оставаться локализованными, вынесены в `docs/` или
   отдельные JSON/YAML файлы.
3. Выполнено `npm run lint:language -- --all` и проверка прошла без ошибок.
