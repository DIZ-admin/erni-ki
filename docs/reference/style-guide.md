# Руководство по стилю документации ERNI-KI

## 1. Emoji

См. `docs/reference/emoji-style-guide.md` для стандартов по категориям и
примеров.

## 2. Admonition Boxes

Используйте только следующие шаблоны:

-**Важно**

> **Важно:**Критически важная информация, которую нельзя пропустить.

-**Предупреждение**

> **ВНИМАНИЕ:**Предупреждение о потенциальных рисках или проблемах.

-**Полезный совет**

> **Совет:**Рекомендация или best practice.

-**Информация**

> ℹ**Информация:**Дополнительные детали или контекст.

## 3. Code Blocks

- Всегда указывайте language tag (`bash, `yaml, ```json).
- Не префиксуйте команды символом `$`; используйте чистый ввод.
- Добавляйте комментарий или ожидаемый результат, если это помогает
  пользователю.

## 4. Списки

- Маркер `-` для bullet points.
- Вложенность — 2 пробела.
- Заголовки разделов в sentence case.

## 5. Таблицы

- Заголовки в**bold**+ разделитель `|---|`.
- Выравнивание по смыслу содержимого (например, `:---:` для центровки чисел).

## 6. Ссылки

- Внутренние ссылки — относительные пути.
- Текст ссылки должен быть осмысленным (не "click here").

## 7. Терминология

- Используйте «сервис» для microservices; «служба» — только для системных служб.
- «Контейнер» — стандартный термин для контейнеров.
- Аббревиатуры — расшифровывайте при первом упоминании.

## 8. Даты и время

- Формат дат: YYYY-MM-DD (ISO 8601).
- Время: 24-часовой формат.
- Помечайте обновление как «Последнее обновление:» при необходимости.

## 9. Кавычки

- `«»` для русской прозы.
- `""` для технических контекстов и CLI-примеров.
