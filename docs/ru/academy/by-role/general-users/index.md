## Практические workflows

### Workflow 1: International Client Communication

**Этапы:**

1. **Draft:** [Write Professional Email](write-professional-email.md) — на вашем
   языке
2. **Translate:** [Translate Document](translate-document.md) — на язык клиента
3. **Review:** Human review перевода
4. **Send:** Отправка email

**Время:** 15 минут (было 45 минут с professional translator) **Результат:**
Professional bilingual communication

---

### Workflow 2: Conference Presentation

**Этапы:**

1. **Structure:** [Prepare Presentation](prepare-presentation.md) — outline и
   slides
2. **Content:** Детализация каждого слайда
3. **Translation:** [Translate Document](translate-document.md) — если
   multi-language event
4. **Practice:** Rehearse с speaker notes

**Время:** 1-2 часа (было 4-6 часов) **Результат:** Ready-to-deliver
presentation

---

### Workflow 3: Team Update Communication

**Этапы:**

1. **Presentation:** [Prepare Presentation](prepare-presentation.md) — slides
   для meeting
2. **Email:** [Write Professional Email](write-professional-email.md) —
   pre-meeting summary
3. **Meeting:** Deliver presentation
4. **Follow-up:** [Write Professional Email](write-professional-email.md) —
   action items

**Время:** 45 минут (было 2 hours) **Результат:** Effective team communication
cycle

---

### Workflow 4: Multi-Language Documentation

**Этапы:**

1. **Create:** Write document на основном языке
2. **Translate:** [Translate Document](translate-document.md) → Language 1
3. **Translate:** [Translate Document](translate-document.md) → Language 2
4. **Review:** Native speakers check
5. **Publish:** Release all versions

**Время:** 2-3 часа (было full day) **Результат:** Multi-language documentation
set

## Best Practices

### 1. Email Writing

```markdown
**Effective email workflow:**

Step 1: Define

- Audience
- Purpose
- Tone
- Key points

Step 2: Generate → [Write Professional Email](write-professional-email.md)

Step 3: Personalize

- Add specific details
- Adjust tone
- Check facts

Step 4: Review

- Grammar check
- Tone check
- Call to action clear?

Step 5: Send
```

---

### 2. Translation Quality

```markdown
**Ensure translation accuracy:**

DO:

- Provide context в промпте
- Specify audience
- Request specific terminology
- Have native speaker review

DON'T:

- Translate confidential data
- Skip human review для important docs
- Ignore cultural differences
- Publish без проверки
```

---

### 3. Presentation Preparation

```markdown
**Maximize presentation impact:**

1. Start with clear goal "What should audience do after?"

2. Use storytelling structure Problem → Journey → Solution

3. Visual > Text Images, diagrams > bullet points

4. Practice delivery Speaker notes → rehearsal

5. Prepare for Q&A Anticipate questions
```

---

### 4. Template Library

```markdown
**Build reusable templates:**

Create library of:

- Email templates (by type)
- Presentation structures
- Translation glossaries

Benefits:

- Consistency
- Speed
- Quality
```

## Чего избегать

### Не делайте:

1. **Send ИИ-generated content без review:**

- Всегда personalize
- Check facts
- Verify tone appropriate

2. **Translate sensitive data:**

- Confidential info
- Personal data (GDPR!)
- Trade secrets

3. **Over-rely на ИИ для creativity:**

- ИИ — starting point
- Add your unique perspective
- Personal touch важен

4. **Ignore cultural context:**

- Especially в translations
- Different markets need different approaches
- ИИ может не знать local customs

## Бонус: Quick Reference

### Email Quick Start

```markdown
**5-Second Email Prompt:**

"Напиши [formal/casual] email [кому] про [тема]. Tone: [professional/friendly].
Length: [brief/detailed]."

Example: "Напиши formal email client про project delay. Tone: professional but
apologetic. Length: brief."
```

---

### Translation Quick Start

```markdown
**5-Second Translation Prompt:**

"Переведи с [язык1] на [язык2]: [текст] Context: [type of document]"

Example: "Переведи с русского на английский: 'Спасибо за встречу...' Context:
business email"
```

---

### Presentation Quick Start

```markdown
**5-Minute Presentation Outline:**

"Создай 10-minute presentation outline: Topic: [тема] Audience: [кто] Goal:
[цель]"

Example: "Создай 10-minute presentation outline: Topic: New product feature
Audience: Sales team Goal: Enable them to demo"
```
