## Безопасность данных

### НИКОГДА не вводите в ИИ:

#### 1. Конфиденциальные данные клиентов

```
 ПЛОХО:
"Проанализируй продажи клиента Siemens за 2024:
Q1: 2.3M EUR, Q2: 1.8M EUR..."

 ХОРОШО:
"Проанализируй продажи Client A за 2024:
Q1: 2.3M, Q2: 1.8M..."
```

#### 2. Персональные данные (PII)

```
 НЕЛЬЗЯ вводить:
- Полные имена с контактами
- Адреса электронной почты
- Телефонные номера
- Адреса проживания
- Номера паспортов/ID
- Медицинские данные
```

#### 3. Коммерческие секреты

```
 НЕЛЬЗЯ:
- Нераскрытые financial figures
- Неанонсированные продукты
- Внутренние стратегии
- Конфиденциальные контракты
- Proprietary algorithms
```

#### 4. Учётные данные и секреты

```
 НИКОГДА:
- Пароли
- API keys
- Токены доступа
- Приватные ключи
- Connection strings
```

### Безопасные практики

#### Анонимизация данных

```
 ХОРОШО:
"Помоги написать email клиенту [Client A], который недоволен задержкой
проекта [Project X] на 2 недели"

Вместо:
"Помоги написать email Thomas Mueller from Siemens, который недоволен..."
```

#### Использование placeholder'ов

```
 ХОРОШО:
Company A, Client B, Project X, User123
Department XYZ, Region EMEA

 ПЛОХО:
Реальные названия, имена, идентификаторы
```

#### Обобщение данных

```
 ХОРОШО:
"Средняя зарплата в IT: 80K-100K"

 ПЛОХО:
"Зарплата John Smith: 87,500 EUR"
```

## Что можно безопасно использовать

- Общие знания из интернета

- Open source код

### Ваша собственная работа

- Ваши идеи и мысли

### Обобщённые данные

- Анонимизированная статистика
- Aggregated metrics
- Общие тренды

## Этические принципы

### 1. Прозрачность

**Правило:** Всегда указывайте, когда контент создан ИИ

```
 ХОРОШО:
"Вот предложения (создано с помощью ИИ, проверено мной)"

 ПЛОХО:
Выдавать ИИ-контент за полностью свой
```

### 2. Ответственность

**Правило:** Вы несёте ответственность за результаты ИИ

```
 ПОМНИТЕ:
- ИИ может ошибаться
- Вы должны проверить результат
- Финальное решение — ваше
- Ответственность лежит на вас, не на ИИ
```

### 3. Честность

**Правило:** Не используйте ИИ для обмана или манипуляций

```
 НЕЛЬЗЯ:
- Создавать фейковый контент
- Манипулировать мнениями
- Plagiarism чужих работ
- Обман клиентов или коллег
```

### 4. Справедливость

**Правило:** Осознавайте потенциальные bias и предвзятости ИИ

```
 ИИ может иметь:
- Культурные предвзятости
- Гендерные стереотипы
- Расовые bias
- Regional preferences

 ВАШ ДОЛГ:
Критически оценивать результаты и корректировать bias
```

## Compliance и правовые аспекты

### GDPR (Общий регламент по защите данных)

#### Основные принципы:

1. **Минимизация данных**

- Используйте минимум персональных данных
- Анонимизируйте где возможно
- Не храните данные дольше необходимого

2. **Согласие**

- Нужно согласие для обработки персональных данных
- ИИ обрабатывает данные → нужно согласие
- Корпоративные данные имеют свои правила

3. **Право на забвение**

- Данные могут быть удалены по запросу
- Не используйте ИИ для постоянного хранения личных данных

### Интеллектуальная собственность

#### Кому принадлежит контент, созданный ИИ?

```
 СЛОЖНЫЙ ВОПРОС:

Юридически неясно во многих юрисдикциях.

 БЕЗОПАСНЫЙ ПОДХОД:
- Считайте ИИ-контент "вспомогательным"
- Всегда вносите свои правки
- Проверяйте на плагиат
- Не полагайтесь полностью на ИИ
```

#### Copyright

```
 ОСТОРОЖНО:
ИИ может воспроизвести copyrighted контент

 ПРОВЕРЯЙТЕ:
- Уникальность текста
- Оригинальность кода
- Наличие заимствований
```

## Защита от рисков

### Риск 1: Утечка данных

**Сценарий:** Вы случайно вставили конфиденциальные данные в промпт

**Защита:**

1. Всегда проверяйте промпт перед отправкой
2. Используйте "поиск и замена" для удаления имён/номеров
3. При сомнении — не отправляйте

### Риск 2: Hallucinations (галлюцинации)

**Сценарий:** ИИ придумал факты, которые выглядят правдоподобно

**Защита:**

1. ВСЕГДА проверяйте факты из других источников
2. Никогда не доверяйте статистике без проверки
3. Перепроверяйте имена, даты, числа
4. Используйте ИИ для идей, не для фактов

### Риск 3: Bias и предвзятость

**Сценарий:** ИИ дал предвзятый или discriminatory ответ

**Защита:**

1. Критически оценивайте ответы
2. Проверяйте на fairness
3. Корректируйте bias вручную
4. Сообщайте о проблемных ответах

### Риск 4: Over-reliance (чрезмерная зависимость)

**Сценарий:** Команда полностью полагается на ИИ без критического мышления

**Защита:**

1. ИИ — это инструмент, не замена мышлению
2. Всегда применяйте профессиональное суждение
3. Обучайте команду критическому подходу
4. Устанавливайте процессы проверки

## Чек-лист безопасности

Перед отправкой запроса в ИИ, проверьте:

- [ ] Нет ли персональных данных (имена, email, телефоны)?
- [ ] Нет ли конфиденциальных данных клиентов?
- [ ] Нет ли коммерческих секретов?
- [ ] Нет ли учётных данных (пароли, токены)?
- [ ] Данные анонимизированы или обобщены?
- [ ] Контекст не нарушает NDA?

После получения ответа, проверьте:

- [ ] Нет ли явных ошибок?
- [ ] Факты проверены из других источников?
- [ ] Нет ли bias или предвзятости?
- [ ] Результат соответствует этике?
- [ ] Можно ли его использовать публично?

## Правильные vs Неправильные примеры

### Пример 1: Анализ данных

```
 НЕПРАВИЛЬНО:
"Проанализируй эти данные клиентов:
- John Smith (john@company.com): 45K revenue
- Jane Doe (jane@firm.com): 67K revenue"

 ПРАВИЛЬНО:
"Проанализируй эти обобщённые данные:
- Client A (B2B, Tech): 45K revenue
- Client B (B2B, Finance): 67K revenue"
```

### Пример 2: Написание кода

```
 НЕПРАВИЛЬНО:
"Вот наш proprietary algorithm для fraud detection:
[внутренний код компании]
Можешь улучшить?"

 ПРАВИЛЬНО:
"Покажи general approach для fraud detection в финансовых транзакциях"
```

### Пример 3: Письмо клиенту

```
 НЕПРАВИЛЬНО:
"Напиши письмо Thomas Mueller (CEO of Siemens Healthcare)
про задержку проекта PHOENIX due to budget cuts"

 ПРАВИЛЬНО:
"Напиши профессиональное письмо CEO клиента
про задержку проекта на 2 недели по техническим причинам"
```

## Ключевые takeaways

1. **Защищайте данные:** Никогда не вводите конфиденциальную информацию
2. **Анонимизируйте:** Используйте placeholders вместо реальных имён
3. **Проверяйте:** Не доверяйте слепо — всегда проверяйте факты
4. **Будьте прозрачны:** Указывайте, когда использовали ИИ
5. **Несите ответственность:** Вы отвечаете за результаты, не ИИ
6. **Критически мыслите:** ИИ — инструмент, не замена вашему суждению
7. **Следуйте политике:** Соблюдайте правила безопасности ERNI
