## Практические workflows

### Workflow 1: Weekly Status Update

**Этапы:**

1. **Сбор данных:** Sprint metrics, team velocity, blockers
2. **Анализ:** [Analyze Metrics](analyze-metrics.md) — trends и insights
3. **Создание отчёта:** [Create Project Report](create-project-report.md) —
   weekly summary
4. **Рассылка:**
   [Write Professional Email](../general-users/write-professional-email.md) —
   stakeholders

**Время:** 30 минут (было 2-3 часа) **Частота:** Weekly

---

### Workflow 2: Sprint Retrospective

**Этапы:**

1. **Подготовка данных:** Velocity, completed stories, bugs
2. **Анализ:** [Analyze Metrics](analyze-metrics.md) — что улучшилось/ухудшилось
3. **Создание summary:** [Create Project Report](create-project-report.md) —
   retrospective report
4. **Meeting prep:**
   [Prepare Presentation](../general-users/prepare-presentation.md) — slides для
   команды

**Время:** 45 минут (было 2 часа) **Частота:** End of sprint (bi-weekly)

---

### Workflow 3: Quarterly Business Review

**Этапы:**

1. **Data collection:** 3 months of metrics
2. **Deep analysis:** [Analyze Metrics](analyze-metrics.md) — trends, forecasts,
   insights
3. **Executive report:** [Create Project Report](create-project-report.md) — QBR
   document
4. **Presentation:**
   [Prepare Presentation](../general-users/prepare-presentation.md) — для
   C-level
5. **Follow-up:**
   [Write Professional Email](../general-users/write-professional-email.md) —
   action items

**Время:** 2-3 часа (было целый день) **Частота:** Quarterly

---

### Workflow 4: Budget Review

**Этапы:**

1. **Financial data:** Actual vs planned spending
2. **Анализ:** [Analyze Metrics](analyze-metrics.md) — burn rate, forecasts
3. **Report:** [Create Project Report](create-project-report.md) — budget status
4. **Stakeholder meeting:**
   [Prepare Presentation](../general-users/prepare-presentation.md)

**Время:** 1 час (было 3-4 часа) **Частота:** Monthly

## Best Practices

### 1. Data Preparation

```markdown
**Before AI analysis:**

DO:

- Clean data (remove duplicates)
- Anonymize sensitive info
- Provide context (what metrics mean)
- Include time periods

DON'T:

- Use raw exports без проверки
- Include PII (personal data)
- Забывать про timezone
```

---

### 2. Report Customization

```markdown
**Tailor report to audience:**

For C-level:

- High-level only
- Business impact focus
- 1 page max
- Visual dashboards

For team:

- Detailed metrics
- Actionable items
- 2-3 pages OK
- Technical depth OK

For stakeholders:

- Progress + blockers
- Timeline implications
- Medium detail
- Clear next steps
```

---

### 3. Regular Cadence

```markdown
**Establish reporting rhythm:**

Daily:

- Quick standup summaries (30 sec generation)

Weekly:

- Status update (5 min generation)
- Team performance (10 min)

Monthly:

- Executive summary (15 min)
- Trend analysis (20 min)

Quarterly:

- Deep dive (30-60 min)
- Strategic planning (45 min)
```

---

### 4. Version Control

```markdown
**Track report evolution:**

- Save prompts that work well
- Keep report templates
- Document data sources
- Version reports (v1.0, v1.1...)
- Archive old reports
```

## Чего избегать

### Не делайте:

1. **Over-reliance на ИИ interpretation:**

- ИИ может неправильно интерпретировать business context
- Всегда verify insights

2. **Публикация без review:**

- Проверяйте цифры
- Verify выводы
- Human approval обязателен

3. **Игнорирование контекста:**

- ИИ не знает ваших project specifics
- Добавляйте business context в промпты

4. **Copy-paste без адаптации:**

- Шаблоны — это starting point
- Адаптируйте под вашу организацию
