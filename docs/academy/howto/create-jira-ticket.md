# Как оформить задачу в Jira с помощью Open WebUI

## Когда использовать

Когда нужно быстро набросать карточку задачи: описание, ожидаемый результат и
критерии приёмки. Сценарий для ручного контроля структуры (без автогенерации).

## Что подготовить

- Заголовок/краткое описание проблемы.
- Шаги воспроизведения, ожидаемый vs фактический результат.
- Окружение (prod/stage/local), логи/скриншоты, дата воспроизведения.

## Шаги

1. Откройте новый диалог и опишите контекст проекта и проблему.
2. Попросите модель сформулировать заголовок, описание и список acceptance
   criteria.
3. Скопируйте черновик в Jira, проверьте термины и даты.
4. Уточните Severity/Impact, labels/компоненты и SLA.

## Примеры промптов

- **Хороший:** «Ты проджект-менеджер. Составь тикет Jira: нужно добавить
  проверку авторизации к API X. Важно: токен, логирование ошибок, уведомление
  при 401. Дай заголовок, описание и 3 acceptance criteria списком».
- **Плохой:** «Надо тикет про авторизацию, придумай сам».

## Готовый промпт

```
Ты — project manager. Оформи баг/фичу для Jira.
Контекст: <факты>. Окружение: <prod/stage>. Дата воспроизведения: <дата>.
Нужно: summary, description (steps, expected vs actual), labels, severity,
3-5 acceptance criteria списком. Не придумывай новые факты.
```

## Чек-лист

- Есть шаги, expected/actual и окружение.
- Acceptance criteria измеримы и проверяемы.
- Нет конфиденциальных данных (пароли, токены, внутренние URL).
