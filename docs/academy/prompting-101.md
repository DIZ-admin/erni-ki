# Промптинг 101

## Что такое промпт

Промпт — это инструкция для модели, которая задает роль, задачу, контекст и
формат ответа. Чем точнее промпт, тем понятнее результат.

## Базовый шаблон

**РОЛЬ → ЗАДАЧА → КОНТЕКСТ → ФОРМАТ**

Пример: "Ты — технический писатель. Нужно подготовить письмо клиенту. Контекст:
отправляем резюме встречи. Формат: 3 абзаца, без маркеров".

Дополните по необходимости: требования к языку (RU/DE/EN), длине текста и тону
голоса (формально/дружелюбно).

## Примеры для офисных задач

- **Письмо клиенту:**
  - "Ты — аккаунт-менеджер. Задача: написать письмо клиенту с итогами встречи.
    Контекст: договорились о тестовом доступе, дедлайн — пятница. Формат:
    приветствие, 3 пункта с итогами, CTA."
- **Резюме созвона:**
  - "Ты — бизнес-аналитик. Задача: сделать резюме 30-минутного митинга.
    Контекст: обсуждали roadmap, приняли 2 решения. Формат: буллеты, блок
    'Риски/Действия'."
- **Улучшение текста:**
  - "Ты — редактор. Задача: улучшить текст письма. Контекст: аудитория — CIO,
    стиль — деловой, без жаргона. Формат: финальная версия + короткие
    рекомендации."
- **Черновик тикета:**
  - "Ты — support-инженер. Задача: подготовить черновик JIRA. Контекст: баг в
    авторизации после обновления, шаги воспроизведения и скриншоты есть. Формат:
    summary, steps to reproduce, expected/actual."
- **Сравнение вариантов:**
  - "Ты — консультант. Задача: сравнить два подхода к деплою. Контекст: k8s vs
    docker-compose. Формат: таблица плюсы/минусы + краткая рекомендация."

## Чек-лист перед отправкой промпта

1. Уточните роль и задачу: зачем модели нужен ответ.
2. Добавьте входные данные: ссылки, выдержки, ключевые цифры.
3. Ограничьте формат: длина, язык ответа, наличие таблиц/списков.
4. Проверьте конфиденциальность: удалите персональные данные и внутренние
   ссылки, если нет разрешения.
5. Зафиксируйте ожидания: что считать готовым результатом (например, "драфт для
   клиента без технического жаргона").
