## Что такое Промпт?

**Промпт (Prompt)** — это ваша инструкция или запрос к ИИ.

### Простая аналогия

Представьте ИИ как очень компетентного, но буквального ассистента:

- Чем точнее вы объясните, тем лучше результат
- Без контекста он может неправильно понять
- С примерами работает лучше
- Может задавать уточняющие вопросы

### Как ИИ "видит" ваш промпт

```
ВЫ ПИШЕТЕ:
"Напиши про Docker"

ИИ ДУМАЕТ:
- О чём именно? (введение? туториал? сравнение?)
- Для кого? (новичок? эксперт?)
- Какой формат? (параграфы? список? код?)
- Сколько деталей? (краткое? подробное?)

РЕЗУЛЬТАТ: Общий, возможно не то что нужно
```

vs

```
ВЫ ПИШЕТЕ:
"Напиши краткое введение в Docker для начинающих
разработчиков (3-4 параграфа). Объясни что это,
зачем нужно, и один простой пример использования."

ИИ ДУМАЕТ:
- Аудитория: новички
- Формат: 3-4 параграфа
- Содержание: что, зачем, пример
- Тон: простой язык

РЕЗУЛЬТАТ: Именно то что нужно!
```

## Анатомия эффективного промпта

Хороший промпт состоит из 4 элементов:

### 1. Роль (Role) — Кто?

Задайте ИИ роль для правильного тона и perspective:

```
"Ты опытный Python разработчик..."
"Как технический писатель, объясни..."
"В роли ментора для новичков..."
```

### 2. Задача (Task) — Что?

Четко опишите, что нужно сделать:

```
"Напиши функцию..."
"Проанализируй и найди ошибки..."
"Сравни два подхода..."
```

### 3. Контекст (Context) — Почему? Где?

Дайте необходимую background информацию:

```
"Мы разрабатываем e-commerce сайт с высокой нагрузкой..."
"Это для презентации менеджерам без технического бэкграунда..."
"Код должен работать на Python 3.9+ без внешних библиотек..."
```

### 4. Формат (Format) — Как?

Укажите желаемую структуру ответа:

```
"Ответь в виде таблицы с тремя колонками..."
"Дай пошаговую инструкцию..."
"Верни результат в JSON формате..."
```

### Полный пример

````markdown
[РОЛЬ] Ты опытный code reviewer.

[ЗАДАЧА] Проанализируй этот Python код и найди потенциальные проблемы.

[КОНТЕКСТ] Это REST API endpoint для production приложения с 100K+
пользователей. Критична безопасность и производительность.

[ФОРМАТ] Структурируй ответ так:

1. Критичные проблемы (security, bugs)
2. Проблемы производительности
3. Улучшения читаемости Для каждой проблемы дай пример исправления.

[КОД] \```python def get_user(user_id): query = f"SELECT \* FROM users WHERE id
= {user_id}" result = db.execute(query) return result \```
````

## 5 основных техник промптинга

### Техника 1: Zero-shot Prompting (Прямой запрос)

**Когда:** Простые, однозначные задачи

**Как:** Просто спросите напрямую

**Пример:**

```
"Переведи этот текст на английский: [текст]"
```

**Плюсы:** Быстро, просто **Минусы:** Может не подходить для сложных задач

---

### Техника 2: Few-shot Learning (Обучение на примерах)

**Когда:** Нужен конкретный формат или стиль

**Как:** Покажите 2-3 примера желаемого результата

**Пример:**

```
Преобразуй user input в структурированный JSON:

Пример 1:
Input: "Встреча с Джоном во вторник в 14:00"
Output: {"type": "meeting", "with": "Джон", "day": "вторник", "time": "14:00"}

Пример 2:
Input: "Позвонить клиенту завтра утром"
Output: {"type": "call", "with": "клиент", "day": "завтра", "time": "утро"}

Теперь преобразуй:
Input: "Презентация проекта в пятницу в 15:30"
```

**Плюсы:** Очень точный результат **Минусы:** Требует подготовки примеров

---

### Техника 3: Chain of Thought (Цепочка рассуждений)

**Когда:** Сложные задачи, требующие логики

**Как:** Попросите показать рассуждения step-by-step

**Пример:**

```
Задача: Компания продала 1200 единиц товара в январе,
что на 20% больше чем в декабре. Сколько продали в декабре?

Реши пошагово, показывая рассуждения на каждом шаге.
```

**Результат:**

```
Шаг 1: Обозначим продажи декабря как X
Шаг 2: Январь = X + 20% от X = X * 1.2
Шаг 3: X * 1.2 = 1200
Шаг 4: X = 1200 / 1.2 = 1000
Ответ: 1000 единиц
```

**Плюсы:** Прозрачная логика, легче проверить **Минусы:** Более длинные ответы

---

### Техника 4: Role Prompting (Ролевой промптинг)

**Когда:** Нужен специфичный tone или expertise

**Как:** Назначьте ИИ конкретную роль

**Примеры:**

```
"Ты senior DevOps инженер с 10 лет опыта в Kubernetes..."
"Как детский психолог, объясни..."
"В роли сатирика напиши..."
```

**Эффект:**

- Меняет vocabulary
- Меняет уровень детализации
- Меняет perspective

---

### Техника 5: Iterative Refinement (Итеративное улучшение)

**Когда:** Первый ответ близок, но не идеален

**Как:** Уточняйте в диалоге

**Пример:**

```
Запрос 1: "Напиши введение в Docker"
[Получили общий ответ]

Запрос 2: "Сделай это короче, максимум 3 параграфа"
[Лучше, но всё равно технично]

Запрос 3: "Упрости язык для не-разработчиков"
[Идеально!]
```

**Плюсы:** Постепенно достигаете идеала **Минусы:** Требует нескольких итераций

## Типичные ошибки промптинга

### Ошибка 1: Слишком расплывчато

```
 ПЛОХО:
"Напиши про машинное обучение"

 ХОРОШО:
"Напиши краткое объяснение supervised learning для
начинающих data scientists (2-3 параграфа с примером)"
```

**Проблема:** ИИ не знает что именно нужно **Решение:** Добавьте конкретики
(аудитория, формат, scope)

---

### Ошибка 2: Отсутствие контекста

```
 ПЛОХО:
"Это хорошая идея?"

 ХОРОШО:
"Контекст: Мы разрабатываем мобильное приложение для
финтех стартапа. Бюджет ограничен.

Идея: Использовать React Native вместо нативной разработки.

Оцени плюсы и минусы для нашей ситуации."
```

**Проблема:** ИИ отвечает в вакууме **Решение:** Опишите ситуацию, constraints,
цели

---

### Ошибка 3: Множественные вопросы в одном промпте

```
 ПЛОХО:
"Что такое Docker, как его установить, какие есть альтернативы,
когда его использовать, и можешь показать пример?"

 ХОРОШО:
"Что такое Docker и когда его стоит использовать?
Дай краткий ответ (3-4 предложения), затем я задам follow-up вопросы."
```

**Проблема:** ИИ может пропустить часть вопросов **Решение:** Один промпт = одна
основная задача

---

### Ошибка 4: Предположение что ИИ "знает" implicit информацию

```
 ПЛОХО:
"Исправь это" [без объяснения что "это"]

 ХОРОШО:
"Вот мой Python код который выдаёт KeyError.
Найди проблему и предложи исправление:
[код]"
```

**Проблема:** ИИ не видит ваш экран, не знает контекста **Решение:** Явно
опишите всё необходимое

---

### Ошибка 5: Игнорирование ограничений ИИ

```
 ПЛОХО:
"Какая погода сейчас в Москве?"

 ХОРОШО:
"Напиши Python скрипт который получает текущую погоду
в Москве используя OpenWeatherMap API"
```

**Проблема:** ИИ не имеет real-time данных **Решение:** Знайте limitations ИИ
(см. [What is AI](../getting-started/what-is-ai.md))

### Упражнение 2: Few-shot Learning

**Задача:** Научите ИИ преобразовывать описания в теги

Создайте 2 примера, затем дайте новое описание:

```
Пример 1:
Описание: "Быстрый и надёжный интернет-магазин"
Теги: #ecommerce #performance #reliability

Пример 2:
Описание: "Мобильное приложение для фитнеса и здоровья"
Теги: #mobile #fitness #health #app

Теперь сгенерируй теги для:
Описание: "Облачная платформа для управления проектами"
```

### Упражнение 3: Chain of Thought

**Задача:** Попросите ИИ решить задачу с показом рассуждений

```
Задача: У команды разработчиков 5 человек. Каждый работает
по 8 часов в день. Проект займёт 200 человеко-часов.
Сколько дней займёт проект?

Реши пошагово, объясняя каждый шаг.
```

## Шаблоны для типовых задач

### Шаблон 1: Написание текста

```
Напиши [тип текста] на тему [тема].

Аудитория: [описание аудитории]
Тон: [формальный/неформальный/технический/и т.д.]
Длина: [количество слов/параграфов]
Формат: [структура]

Ключевые пункты:
- [пункт 1]
- [пункт 2]
```

### Шаблон 2: Анализ кода

````
Проанализируй этот [язык] код:

Фокус:
1. [аспект 1, например: безопасность]
2. [аспект 2, например: производительность]
3. [аспект 3, например: читаемость]

Код:
\```[язык]
[ваш код]
\```

Для каждой проблемы:
- Опиши issue
- Оцени severity (high/medium/low)
- Предложи исправление с примером
````

### Шаблон 3: Обучение/Объяснение

```
Объясни [концепция] для [уровень аудитории].

Включи:
1. Простое определение (1-2 предложения)
2. Зачем это нужно (практическая польза)
3. Пример из реальной жизни или код
4. Распространённые заблуждения

Используй аналогии где возможно.
```

## Debugging промптов

Если ответ не такой как ожидали:

### Чек-лист диагностики:

1. **Достаточно ли конкретики?**

- Добавьте детали о формате, стиле, длине

2. **Дан ли контекст?**

- Объясните ситуацию, аудиторию, constraints

3. **Понятна ли задача?**

- Переформулируйте проще и яснее

4. **Правильная ли модель?**

- GPT-4o для креативности
- Claude для точности
- Llama для скорости

5. **Нужны ли примеры?**

- Добавьте few-shot examples

### Стратегия улучшения:

```
Итерация 1: Базовый промпт
↓
Оцените результат
↓
Итерация 2: Добавьте конкретику
↓
Лучше? Если нет →
↓
Итерация 3: Добавьте примеры
↓
Лучше? Если нет →
↓
Итерация 4: Смените модель или подход
```

## Pro Tips

### Tip 1: Используйте Markdown форматирование

````markdown
Улучшает читаемость для ИИ:

## Заголовки

- Списки **Акценты** \```code blocks\```
````

### Tip 2: Разделяйте инструкции и данные

```
Инструкция:
Найди грамматические ошибки в тексте ниже.

---

Текст:
[ваш текст здесь]
```

### Tip 3: Будьте explicit с границами

```
Напиши РОВНО 3 параграфа.
Используй ТОЛЬКО информацию из предоставленного документа.
НЕ добавляй дополнительных библиотек.
```

### Tip 4: Используйте constraints

```
Ограничения:
- Максимум 100 слов
- Только Python standard library
- Без использования AI jargon
```
